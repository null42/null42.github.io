# Mission: UPS 控制方向入职训练

## 学习者画像

- 电气工程专业背景。
- 学过 UPS 相关专业课，但很多基础已经遗忘或掌握不牢。
- 有电机控制经验，希望把这部分经验迁移到 UPS 电源控制工作中。
- 入职后大概率偏向 UPS 控制相关软件，但仍需要理解硬件拓扑、采样调理、嵌入式、保护逻辑、通信和系统工程。
- 英文阅读能力一般，学习资料必须以中文解释为主，同时保留关键英文术语，方便后续看代码、芯片手册、应用笔记和公司文档。

## 一个月目标

在入职后的第一个月内建立足够基础，能够读懂 UPS 控制相关代码、控制框图和仿真模型；能够用 Python 或 MATLAB/Simulink 做基础验证；能够带着明确问题向导师请教；能够参与非关键模块的学习、调试和辅助开发。

## 长期目标

完成训练体系后，应逐步具备三类能力：

- 能自行推导 UPS 相关拓扑的基本工作原理，包括 Boost、功率因数校正（power factor correction / PFC）、LLC 谐振变换器（inductor-inductor-capacitor resonant converter / LLC）、维也纳整流器（Vienna rectifier）和三电平逆变器（three-level inverter）。
- 能较顺畅阅读带有部分注释的完整 C 语言工程代码，理解控制中断、采样、脉宽调制（pulse-width modulation / PWM）、保护、状态机、通信和参数管理之间的关系。
- 能自行实现一个具备基本功能的简化 UPS 软件骨架，包括启动、采样、控制计算、保护判断、模式切换、故障锁存、通信接口和日志记录。

## 教学语言要求

1. **中文主讲**：概念解释、物理直觉、推导过程、仿真现象和复盘都优先使用中文。
2. **中英对照**：第一次出现重要术语时使用“中文名（English term / abbreviation）”格式，例如“占空比（duty cycle / D）”“连续导通模式（continuous conduction mode / CCM）”。
3. **保留工程英文**：代码变量、命令、文件名、芯片手册中的固定术语可以保留英文，但需要在正文中解释含义。
4. **缩写必须展开**：首次出现 PFC、PLL、PWM、ADC、ISR、RTOS、BMS、DCM、CCM、SVPWM、PR、PI 等缩写时，必须写出英文全称和中文含义。
5. **不直接堆英文资料**：引用英文资料时，需要先用中文说明它解决什么问题、应该重点看哪一部分。

## 训练原则

1. 每个概念都必须按“定义、为什么需要、物理直觉、数学形式、仿真观察、固件落地、常见误解”解释清楚。
2. 理论必须配套 Python 或 MATLAB/Simulink 仿真。
3. 旧生成课程只是归档素材，不再作为主学习路径。
4. 向导师请教前必须准备具体产物，例如仿真波形、参数、代码片段或明确问题。
5. 公司内部代码、未公开产品方案、客户项目参数、内部测试数据等保密信息不能写入本学习仓库。

## 教程交付格式要求

1. Markdown 可以作为源稿、速查资料和版本维护格式，但不能作为面向学习者的唯一教程入口。
2. 按 teach skill 的要求，正式教程必须提供 HTML 页面，保存在 `lessons/` 目录中。
3. HTML 教程必须中文主讲、中英术语对照，并包含清晰目录、分节讲解、代码命令、仿真任务和验收标准。
4. HTML 教程必须能直接在浏览器中打开阅读，不依赖额外构建步骤。
5. 如果 Markdown 概念文档与 HTML 教程同时存在，HTML 教程是学习入口，Markdown 是维护源稿或补充材料。
6. 新增或修改理论内容时，必须同步检查对应 HTML 教程是否需要更新。

## 仿真教学工具要求

1. 仿真脚本默认应在生成图片后自动打开结果图，不能只把图片保存到目录里让学习者自己找。
2. 如果因为系统环境无法自动打开图片，脚本必须清楚打印图片的完整路径。
3. 每张仿真图必须直接包含关键参数，例如输入电压、占空比、电感、电容、负载、开关频率、仿真时长。
4. 每条曲线必须有图例或明确标签，不能让学习者猜哪条曲线代表什么。
5. 对比多组参数时，文件名、图标题或图内参数框必须说明当前参数条件。
6. 仿真图应服务理解，优先清楚表达物理现象，不追求花哨样式。

## 成功标准

- 能从物理直觉到固件实现解释 Boost、PFC、PLL、逆变器等核心控制概念。
- 能搭建和解释基础 Python 或 MATLAB/Simulink 模型。
- 能区分控制环路问题、采样问题、保护问题、状态机问题和通信问题。
- 能基于仿真或代码证据准备清晰的导师问题。
- 能看到英文工程术语时知道对应中文含义和实际用途。
- 能阅读简化但完整的 UPS C 工程骨架，知道每个模块为什么存在、何时被调用、输入输出是什么。
- 能实现一个基本 UPS 软件原型，哪怕控制对象先用 Python 或仿真模型替代真实硬件。
